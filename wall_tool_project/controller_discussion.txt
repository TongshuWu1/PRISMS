Active Controller
-----------------
The selected controller is:
  control_law = "tool_head_nmpc"

The controller is a direct nonlinear MPC. It optimizes tool-head motion,
payload tilt, cable support, drone thrust, and reel command over a finite
horizon.

State:
  x = [p_x, p_z, v_x, v_z, phi, omega, l]^T

Command:
  u = [F_L, F_R, T, ldot_cmd]^T

The MPC receives the sampled desired path horizon:
  p_des,0 ... p_des,N
  v_des,0 ... v_des,N

Only the direct nonlinear MPC path is kept in the active project.

Cable Model
-----------
The active branch uses an inextensible unilateral steel cable:
  ||anchor - cable_mount(p, phi)|| <= l
  T >= 0

The cable can pull when taut, cannot push, and cannot provide more than 100%
vertical support:
  T * e_c,z <= m g

Tilt Policy
-----------
Body pose is not a tracking task. Tilt is an internal optimization variable.
The solver uses tilt only when it improves path tracking or reduces future
drone effort, subject to the configured attitude bound.

Objective Priority
------------------
Hard constraints:
  wall bounds, actuator limits, cable tension limit, cable support limit,
  cable length bounds, slack tolerance, attitude bounds, reel speed and
  acceleration limits, numerical stability.

Primary objective:
  minimize tool-head tracking error over the horizon.

Secondary objectives:
  smooth velocity tracking, low drone effort, useful cable support, smooth reel
  command, smooth input changes, small angular rate, and no unnecessary tilt.
